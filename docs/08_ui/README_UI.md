# MontrixBot UI / UX — Канон (документация)

Статус проекта: PRODUCT-MODE ACTIVE  
Статус UI: read-only к REAL

Этот документ является **единственным каноническим описанием UI MontrixBot**.
Он фиксирует назначение экранов, их роль в архитектуре SIM / Gate / REAL
и правила взаимодействия UI с runtime.

Документ **не является роадмапом**,  
**не требует изменений runtime**,  
**не даёт разрешений на реализацию**.

---

## 1. Базовая философия (канон)

- **SIM = интеллект без власти**
- **Gate = проверка права действовать (evidence-based)**
- **REAL = исполнение без интеллекта**
- **UI = наблюдение, объяснение, аудит**

UI никогда не управляет REAL напрямую.

---

## 2. Роль UI в системе

UI предназначен для:
- понимания текущего состояния системы;
- объяснения причин блокировок и решений;
- анализа истории решений и событий;
- инспекции стратегий и политик.

UI **не предназначен** для:
- торговли;
- ручного управления ордерами;
- обхода Gate;
- скрытых управляющих действий.

---

## 3. Канонический набор экранов

### 3.1 Main
Назначение: быстро понять **что происходит с системой сейчас**.

Отображает:
- FSM
- MODE
- Gate decision
- WHY NOT
- возраст статуса (age)
- системные уведомления

Источники данных:
- `runtime/status.json`
- `runtime/events.jsonl`

---

### 3.2 Strategies
Назначение: декларативная инспекция стратегий.

Отображает:
- список стратегий;
- статусы и веса;
- параметры (overview / entry / exit / risk);
- состояние (active / frozen).

Запрещено:
- торговые действия;
- прямое включение исполнения.

---

### 3.3 Explain
Назначение: объяснимость.

Отображает:
- WHY NOT;
- последнее решение Gate (reasons + evidence);
- последнее решение SIM (recommendation + confidence).

---

### 3.4 Journal
Назначение: аудит и расследование.

Отображает:
- SIM Decision Journal;
- System Events;
- фильтры по времени и типам.

Журнал append-only, UI read-only.

---

### 3.5 Settings
Назначение: инспекция политик и лимитов.

Отображает:
- режим системы;
- risk limits;
- AUTO-ограничения;
- HARD_STOP / safety.

Settings не являются control-surface.

---

## 4. Trade Viewer (отдельный экран)

Назначение:
- реконструкция одной сделки;
- timeline ENTRY / STOP / EXIT;
- контекст SIM и Gate.

Запрещено:
- BUY / SELL;
- управление позицией.

---

## 5. AUTO — зафиксированные ограничения

AUTO может:
- снижать веса;
- замораживать стратегии.

AUTO не может:
- повышать веса;
- включать стратегии;
- снимать HARD_STOP.

---

## 6. Legacy / reference UI

Legacy UI используется **только как справка**.

Файлы:
- `legacy/LEGACY_UI_INDEX.md`
- `legacy/LEGACY_UI_PATTERNS.md`
- `legacy/LEGACY_MOONBOT_UI.md`
- `legacy/aegiron_ui_prototype.html`

Legacy не является каноном и не может переопределять этот документ.

---

## 7. Контракты данных UI

Связь UI ↔ runtime описана в:
- `UI_DATA_CONTRACTS.md`

Этот файл описывает **как UI читает факты**,  
но не задаёт формат runtime.

---

## 8. Итог

- UI read-only к REAL.
- Explain и Journal обязательны.
- Legacy изолирован.
- Runtime не затрагивается.
- Роадмапы и governance не переопределяются.

Этот документ — верхний уровень UI-канона.
